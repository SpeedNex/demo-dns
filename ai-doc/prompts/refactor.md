# Refactoring 重构提示词

> 用于 AI 执行代码重构任务

## 重构原则

1. **保持行为不变**：重构不改变代码的外部行为
2. **小步前进**：每次重构只做一件事
3. **随时可停止**：重构后的代码必须时刻处于可运行状态
4. **配套测试**：重构后必须运行测试验证

---

## 重构准备

1. 读取 `project-doc/01-ARCHITECTURE.md` 了解架构设计
2. 读取 `rules/coding.md` 了解编码规范
3. 读取 `rules/naming.md` 了解命名规范
4. 确认代码现有功能

---

## 常见重构类型

### 1. 函数重构

| 重构类型 | 说明 |
|---------|------|
| 提取函数 | 超过 50 行的函数拆分为小函数 |
| 合并函数 | 功能相同的短函数合并 |
| 参数对象 | 超过 5 个参数用数组替代 |
| 移除参数 | 不再使用的参数必须删除 |

### 2. 变量重构

| 重构类型 | 说明 |
|---------|------|
| 重命名 | 语义不清的变量改为清晰名称 |
| 提取变量 | 复杂表达式结果提取为变量 |
| 内联变量 | 只用一次的变量内联 |
| 魔术数字 | 硬编码数字改为命名常量 |

### 3. 类重构

| 重构类型 | 说明 |
|---------|------|
| 提取类 | 职责过多的类拆分 |
| 合并类 | 职责过少的类合并 |
| 移动方法 | 方法移动到更合适的类 |
| 提取接口 | 公共行为提取为接口 |

### 4. 条件表达式重构

| 重构类型 | 说明 |
|---------|------|
| 提取条件 | 复杂条件判断提取为函数 |
| 合并条件 | 多个相同目的条件合并 |
| 卫语句 | 用 early return 消除嵌套 |

## 重构检查清单

### 重构前
- [ ] 理解代码现有功能
- [ ] 确认有测试覆盖（或手动验证方案）
- [ ] 记录需要重构的部分

### 重构中
- [ ] 每次只做一个类型的重构
- [ ] 重构后立即运行测试
- [ ] 保持代码可编译/可运行

### 重构后
- [ ] 功能行为未改变
- [ ] 测试全部通过
- [ ] 无新增警告
- [ ] 代码符合规范

## 注意事项

### 可以做的
- ✅ 提取重复代码为公共函数
- ✅ 重命名语义不清的变量
- ✅ 拆分过长的函数
- ✅ 用卫语句消除嵌套

### 不可以做的
- ❌ 不改变代码行为
- ❌ 不在这此重构中同时做功能修改
- ❌ 不删除看似"无用"但实际有用的代码（除非确认）
- ❌ 不引入新的依赖
