# Bug Fix 修复提示词

> 用于 AI 执行 Bug 修复任务

## 修复原则

1. **只修改与 Bug 直接相关的代码**
2. **不破坏现有行为**
3. **不改变公共 API**
4. **不降低安全性**
5. **修复根因，不掩盖现象**

## 修复流程

### 1. 问题分析

- 复现问题（提供复现步骤）
- 定位问题代码
- 分析根因

### 2. 修复实施

- 最小化修改
- 显式处理异常路径
- 确保资源释放
- 不引入新的兜底值

### 3. 验证

- 问题是否修复
- 是否破坏现有功能
- 是否引入新问题

## 修复后检查清单

- [ ] 问题根因已修复，非表象掩盖
- [ ] 异常路径已处理（null/false/空值/异常）
- [ ] 资源已释放（文件/连接/锁）
- [ ] 外部接口返回值已检查
- [ ] 未删除已有错误处理
- [ ] 未降低安全性
- [ ] 未关闭 SSL 校验
- [ ] 未改变返回值语义

## 输出格式

```
## Bug 修复报告

### 问题描述
[描述问题]

### 根因分析
[分析根因]

### 修复方案
[描述如何修复]

### 修改文件
- xxx.php

### 检查清单
- [x] 问题已修复
- [x] 现有功能未破坏
- [x] 安全性未降低
...
```
