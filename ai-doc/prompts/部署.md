二、geodns 部署位置与方式
2.1 部署在哪里
geodns 是入口调度器，负责"用户该访问哪个 resolver 节点"。它的部署位置分两种情况：

场景	部署位置	作用
本地开发	同一台机器，:5354	与 resolver 同机，验证调度逻辑
生产环境	独立服务器/容器，公网可达	作为 dns.ocerdns.local 的权威 DNS 服务器，根据用户地域返回最近 resolver IP
架构说明（来自 01-ARCHITECTURE.md）：

GeoDNS 的职责是入口寻址，不是每次 DNS 递归查询的中间代理。

五、关键注意事项
TLS 终止：dns-resolver 只监听 HTTP（main.go:L228-233），生产环境必须由 Nginx/Caddy 在前面提供 HTTPS。浏览器 DoH 要求 HTTPS。

Profile UID 格式：isValidProfileUID 只允许 [a-zA-Z0-9_]、长度 4-64 字符。profile_uid 必须与 portal-web 中创建的 Profile ID 一致。

geodns 不是必需的：本地开发/单节点部署时，geodns 可跳过。它只在多节点、多地部署时用于地域调度。

自签名证书：本地开发用自签名证书时，浏览器会提示不安全。可以在 Chrome 中访问 chrome://flags/#allow-insecure-localhost 开启允许，或将自签名证书导入系统信任链。

一键启动脚本：start-all.sh 自动启动全部 3 个服务，Pid 文件在 .run/，日志在 logs/。
